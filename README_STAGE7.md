# Stage 7 — Empirical Realignment and Final Research Delivery

Stage 7 reconciles the repository with the original research design.

The repository may contain a technically complete Stage 4–6 pipeline while still lacking the evidence required to answer the original research question. Stage 7 therefore begins with an alignment audit rather than assuming the study is ready for final submission.

The original empirical design requires two substantive components:

1. computational content analysis across substantively different source types;
2. publication-aligned market-quality analysis for markets cited in those texts.

Event-window analysis remains optional and is included only when timestamps and time-series data pass explicit feasibility gates.

Possible Stage 7 outcomes:

- `finalization_ready`
- `empirical_recovery_required`
- `design_revision_required`

A successful software run is not equivalent to a successful empirical study.
